# React 오케스트레이터

react 하네스의 6단계 에이전트 파이프라인을 조율하는 엔트리포인트다.
이슈나 요청을 받으면 아래 파이프라인을 순서대로 실행한다.

## 파이프라인

```
이슈/요청
    │
    ▼
r-analyst       (1단계) 요구사항 분석
    │  요구사항 문서
    ▼
r-architect     (2단계) 기술 청사진 설계
    │  청사진
    ▼
r-inspector     (3단계) 현재 코드 분석
    │  코드 현황 보고서
    ▼
r-planner       (4단계) 기술 변경 계획 수립  ← 사용자 승인 체크포인트
    │  실행 계획 (승인 후)
    ▼
r-executor      (5단계) 계획 실행
    │  변경된 코드
    ▼
r-verifier      (6단계) 요구사항 충족 검증
    │
    ▼
✅ 승인 / ❌ 재작업
```

## 사용자 승인 체크포인트

파이프라인은 **4단계(r-planner)** 완료 후 반드시 멈추고 사용자 승인을 받는다.

```
[r-planner 완료]
실행 계획을 준비했습니다. 검토 후 실행을 승인해주세요.
[계획 내용 표시]
→ 승인하면 r-executor를 시작합니다.
```

## 에이전트별 역할 요약

| 에이전트 | 단계 | 입력 | 출력 |
|----------|------|------|------|
| `r-analyst` | 1 | 이슈/요청 | 요구사항 문서 |
| `r-architect` | 2 | 요구사항 문서 | 기술 청사진 |
| `r-inspector` | 3 | 청사진 + 코드 | 코드 현황 보고서 |
| `r-planner` | 4 | 청사진 + 현황 | 실행 계획 |
| `r-executor` | 5 | 실행 계획 | 변경 코드 |
| `r-verifier` | 6 | 변경 코드 + 요구사항 | 검증 보고서 |

## 부분 실행

전체 파이프라인이 아닌 특정 단계만 실행할 수도 있다.

```
# 요구사항 분석만
r-analyst로 이 이슈 분석해줘: [이슈 내용]

# 코드 분석만
r-inspector로 src/app/home 분석해줘

# 리뷰만
r-review 스킬로 이 코드 리뷰해줘

# 특정 단계부터 재시작
r-planner부터 다시 시작해줘 (청사진은 이미 있음)
```

## 재작업 흐름

`r-verifier`가 재작업을 요청하면 문제 유형에 따라 되돌아간다.

```
코드 수정으로 해결 가능     → r-executor 재실행
설계 변경 필요             → r-architect 재실행
요구사항 재확인 필요        → r-analyst 재실행
```
